## Cases → featured + /cases при 7+ проектах

**Триггер:** когда в массиве `cases` в [Cases.tsx](../components/sections/Cases.tsx) накопится 7 или больше записей.

**Что делать:**

1. На главной секция Cases показывает **3 featured-кейса** под текущий оффер.
2. Ниже карточек — ссылка «Все кейсы →» на отдельную страницу `/cases`.
3. Создать `app/(marketing)/cases/page.tsx` с полным списком (все записи из массива в той же верстке, `grid-cols-3`).
4. В `app/sitemap.ts` добавить `/cases`.
5. Schema.org — Service-блоки остаются на главной, на /cases опционально добавить ItemList или ограничиться Organization в layout.

**Зачем:** главная не раздувается вертикально, клиент сразу видит сильнейшее портфолио, остальное доступно одним кликом. Глубокое SEO по слову «кейсы» уходит на /cases.

**Параллельно при 10+ кейсов:** внутри `/cases` добавить фильтр-табы «Все / SaaS / Боты / Mobile / Автоматизации».

До порога 6 проектов — ничего не трогать, просто добавлять в массив `cases`, сетка `lg:grid-cols-3` сама выстраивается в 3+3.

## MailKit — идея нового SaaS

**Триггер:** когда текущие проекты Vibecraft стабилизируются и освободится ресурс на новый продукт.

Бриф: [mailkit-brief.md](mailkit-brief.md).

Суть: автоматизация end-to-end настройки `hello@домен.kz` (Cloudflare Email Routing + SMTP-провайдер + Gmail Send-As + подпись) через OAuth и API за 5 минут. Pay-per-setup $5 + опциональная подписка на мониторинг доставляемости.

Зачем записано: идея родилась 2026-04-19 из личной боли Boris при настройке почты для Vibecraft. Validated pain — Fiverr gigs за $10 one-time на ту же задачу уже существуют, productized решения нет.

Следующий шаг (когда готовы): валидация на X/Indie Hackers — опрос «заплатил бы $5 пропустить setup?», цель 50+ yes.

## Аудит/оптимизация сайтов как отдельный фриланс-сервис

**Триггер:** только если докажем рентабельность. Внесено по идее Boris 2026-04-22 после удаления функции аудита из Vibecraft (она размывала оффер по разработке).

Идея: продавать аудит + починку сайта как **отдельный сервис на Fiverr / Kwork / Upwork / отдельный лендинг**, не на vibecraft.kz. Логика: люди ищут целево «оптимизация сайта», «починка сайта» — им проще сгрузить задачу и заплатить, чем гонять PSI вручную → копаться в советах ChatGPT → гуглить как чинить. В итоге всё равно идут к фрилансеру или в студию.

**Гипотеза цены:** $20-50 за разовый аудит с PDF-отчётом + апсейл «починю за $X».

**Сомнения по рентабельности (требуют валидации перед запуском):**
- Маржа: $20-50 за аудит × ~10-50 заказов/мес = $200-2500/мес. Альтернативная стоимость часа Boris (Vibecraft = от $50/час) — может быть невыгодно если поток слабый.
- Конкуренция на Fiverr/Upwork высокая, демпинг от агентств из Индии/Пакистана.
- AI генерирует похожий отчёт за 5 минут — клиент может сделать сам, если знает что искать. Платит только за «не хочу разбираться».
- Нужен трафик: Fiverr/Upwork даёт первый, но рейтинг растёт месяцами.

**Перед стартом:**
1. Прайс-чек: посмотреть топ-10 услуг «website audit / website speed audit» на Fiverr — цены, рейтинги, активные заказы.
2. Прикинуть unit-экономику честно: сколько часов на 1 заказ × ставка vs цена.
3. Если рентабельность подтверждается — собрать MVP сервиса (отдельный лендинг + автоматизация PSI + ручная работа на починку), запустить тест на Fiverr.
4. Если нет — закрыть идею.
