# Vibecraft — как делать страницы услуг под SEO + GEO (гайд)

**Дата:** 2026-07-07
**Цель:** страницы услуг `/uslugi/[slug]` должны сами приводить трафик из Google.kz И из AI-поиска (ChatGPT, Perplexity, Google AI Overviews) — БЕЗ опоры на блог-статьи. Каждая страница = одна услуга = один head-ключ из прогона (docs/seo-blog-plan.md).

Основано на research актуальных практик 2026 (источники внизу).

---

## 1. Принцип: SEO и GEO — разные сигналы, нужны оба
- **SEO** (Google-выдача): ранжирование по ключу, E-E-A-T, ссылки, Core Web Vitals.
- **GEO** (AI-поиск): AI-движки уже дают 12-18% информационных запросов (Q1 2026), традиционный поиск падает. AI не ранжирует список — он ЦИТИРУЕТ. Нужно попасть в цитату.
- Сильный GEO стоит на сильном SEO-фундаменте. Делаем обе стороны на одной странице.

## 2. Ключ на страницу — один
- 1 страница = 1 главный head-ключ + 2-3 близких (не stuffing). Ключи из прогона, привязаны в `lib/services-pages.ts`:
  - chat-boty → «разработка чат бота» (1392)
  - ai-agenty → «ии агент для бизнеса» (1016)
  - veb-prilozheniya → «разработка веб приложения» (4356)
  - ai-sajty → «создание сайта под ключ» (1406)
  - avtomatizacii → «автоматизация бизнеса» (11202)
  - mobilnye-prilozheniya → «разработка мобильного приложения» (5981)
- Ключ в: `<title>` (в начале), H1, первые 100 слов, 1-2 H2, meta description, slug, alt картинок.

## 3. GEO-правило №1: первые 150-200 слов = ПРЯМОЙ ответ
- AI с реал-тайм поиском (Perplexity, AI Overviews) оценивают релевантность по НАЧАЛУ страницы. Первые 150-200 слов должны СРАЗУ и ПОЛНО ответить на запрос («Разработка чат-ботов в Казахстане — это... сроки... цена... что входит»), а не подводить к ответу.
- Пиши так, чтобы AI мог вырезать абзац как готовую цитату: конкретные факты, цифры, сроки, цена ₸.

## 4. Структура страницы (секции по порядку)
1. **Hero (above-the-fold):** H1 с ключом + КЗ + 1-2 предложения что это и для кого + цена/срок + CTA-кнопка. Все видно без скролла (10 секунд на понимание — иначе −50% конверсии).
2. **Прямой ответ (GEO-блок):** 150-200 слов — что за услуга, что входит, сроки, цена ₸, для кого. Это блок под AI-цитату.
3. **Что входит:** буллеты конкретики (не вода).
4. **Для кого / отрасли:** сценарии применения (салон, клиника, магазин, доставка) — ловит отраслевой long-tail + AI-запросы «X для [отрасли]».
5. **Цена и сроки:** таблица/блок с ₸ и неделями. Прозрачность = доверие (КЗ-аудитория боится скрытых цен).
6. **Почему Vibecraft / E-E-A-T:** кто делает (Борис, AI Native Founder), стек (Claude Code), код клиенту, гарантия правок. First-hand expertise.
7. **FAQ:** 4-6 вопросов-ответов (сколько стоит, сроки, что нужно от меня, код мой?). Критично для SEO-сниппетов И для AI-цитирования.
8. **CTA-блок:** форма/кнопка «обсудить проект» + ссылка на AI-калькулятор.

## 5. Тексты
- Живой человеческий язык, без LLM-клише и без кучи тире (—). КЗ-якоря органично: Kaspi, Halyk, ₸, Алматы, Астана, ТОО, ИП, ЭСФ, 2ГИС.
- Конкретика вместо воды: «за 2 недели», «от 1 500 000 ₸», «код целиком у вас».
- E-E-A-T: писать от лица того кто реально делал (опыт, конкретные решения), не абстрактно.
- «Вы/ваш» — официальный голос (лендинг), НЕ «ты». Никогда Е/е — только Е/е.

## 6. Картинки
- Каждой картинке осмысленный alt С ключом/контекстом (не «image1»). Alt читают и Google, и AI.
- Формат WebP, ширина ≤1200px, вес <150KB — под LCP (Core Web Vitals влияют на ранг И конверсию).
- Реальные скрины работ/интерфейсов > стоковой абстракции (E-E-A-T, доверие). Абстрактная геометрия не привлекает.
- Первый экран (LCP-картинка) — приоритетная загрузка.

## 7. Шрифты и дизайн
- НЕ изобретать — использовать существующую дизайн-систему проекта: компоненты `Container`, `SectionHeading`, токены `text-foreground`/`text-muted`/`text-subtle`/`accent-text`, `bg-card`/`bg-surface`/`border-border`, `bg-gradient-accent`, `rounded-2xl`. Шрифт — тот же что на сайте (font-mono для акцентов, sans для текста).
- Темная тема сайта = единый вид со всеми страницами. Страница услуги должна выглядеть как часть сайта, не как чужой лендинг.

## 8. Schema (structured data) — критично для AI
Схема дает до 3.2x цитирований в AI Overviews и 2.5x попадания в AI-ответы. На каждой странице услуги:
- **Service schema** (@type: Service): name (= услуга), description, provider (Organization Vibecraft), areaServed (Казахстан/Алматы/Астана), offers (цена ₸).
- **FAQPage schema** — из блока FAQ (дает сниппеты + AI-цитаты).
- **BreadcrumbList** — Главная → Услуги → [Услуга].
- **Organization + Person** (Борис Комаров) — на уровне сайта, для E-E-A-T. Person делает экспертизу машиночитаемой.
- КРИТИЧНО: данные в schema должны ТОЧНО совпадать с текстом на странице (цена, услуга). Расхождение → Google обесценивает всю разметку.

## 9. Перелинковка (усиливает страницы без статей)
- Каждая страница услуги линкует ВВЕРХ на главную и ВБОК на related-услуги (боты ↔ AI-агенты, веб ↔ AI-сайты).
- Главная (карточки Services) → страницы услуг.
- Позже блог long-tail → страница услуги (качает head). Но страницы работают на трафик и БЕЗ статей — через прямой ключ + schema + GEO-ответ.
- Все страницы в `sitemap.ts`, дропдаун «Услуги» в навбаре.

## 10. Технический чеклист
- Уникальные `<title>` (≤60) и meta description (140-160) на каждую, ключ в начале.
- `canonical` на себя.
- Mobile-first, Core Web Vitals (LCP/INP/CLS) в зеленой зоне.
- OpenGraph + Twitter card.
- В `sitemap.ts` + `robots.ts` открыты.
- llms.txt на сайте (помогает AI-движкам понять структуру) — проверить/добавить.

## Про сам сайт
Те же принципы применимы к главной и остальным страницам: первые слова = суть, schema (Organization/Person), КЗ-якоря, E-E-A-T, скорость, перелинковка. Сайт целиком должен работать на трафик, а не только блог.
