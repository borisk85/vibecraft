- **Не переключаться между задачами.** Если Boris в процессе говорит «еще надо вот это» — это для TODO, не для немедленного переключения. Всегда доделать текущую задачу до конца, потом взять следующую.
- **Списки максимум 3-5 пунктов.** Никаких «Принципы / Why / Как применять / Альтернативы / Итог».
- **Заголовки `##` в чате — почти всегда лишние.**
- **Не объяснять очевидное.** Не говорить «вставь код и нажми Deploy», «открой ссылку» — знает и делал 100 раз.
- **После каждого фикса — короткое объяснение простым языком:** что было не так → что изменилось → как работает теперь. Без названий функций в стиле «добавил _foo() в bar.ts».
- **Формат «одного шага» при тестировании флоу:** `ШАГ N: [одна команда]` + `ЖДУ: [что Boris ответит]`.

## Дизайн-правила (обязательные, не отклоняться)

```
bg-primary:    #0A0A0A   (основной фон)
bg-secondary:  #111111   (вторичный фон)
bg-card:       #1A1A1A   (карточки, поверхности)

text-primary:   #EDEDED

accent:        #8B5CF6   (фиолетовый electric)
accent-pink:   #EC4899   (для градиента)

warning:       #F59E0B
error:         #EF4444
```

Градиент: `linear-gradient(135deg, #8B5CF6 0%, #EC4899 100%)`.

### Использование градиента — СТРОГО дозированно

Градиент используется только в **3-4 местах на всем лендинге**:
1. Ключевое слово/фраза в H1 Hero (через `bg-clip-text text-transparent`).
2. Primary CTA-кнопка (фон).
3. Одна акцентная blob-подсветка за Hero (статичная — `absolute div` с `bg-gradient-accent`, `blur-[120px]`, `opacity-30`, БЕЗ движения).
4. Возможно — иконки активных состояний.

Везде остальное — монохромный фиолетовый `#8B5CF6` как обычный акцент (ссылки, hover-рамок карточек, подчеркивания). Градиент на каждом блоке = бюджетный лендинг.

### Типографика
- Заголовки (h1-h3): Inter, вес 600-700, tracking `-0.02em`, h1 — 56-80px на десктопе.
- Текст: Inter, вес 400-500, 16-18px, line-height 1.6.
- Акценты/код/цифры: JetBrains Mono.

### Компоненты
- Кнопки: радиус 8-12px, primary с градиентом, secondary — outline с тонкой рамкой. Hover: subtle glow или легкий lift (`translateY -2px`).
- Карточки услуг: фон `#1A1A1A`, рамка 1px `#2A2A2A`, радиус 16px, hover → рамка подсвечивается `#8B5CF6`.
- Формы: underline-стиль или тонкие рамки.
- Разделители блоков: НЕ горизонтальные линии. Разделение через padding 80-120px сверху/снизу каждого блока.

### Анимации (Framer Motion)
- Fade-in + slight translate при появлении блоков (scroll-triggered).
- Плавный hover на карточках (scale 1.02, 200ms ease-out).
- Градиентные тексты для ключевых заголовков.
- Никаких глитчей, parallax, движущихся blob'ов — признак колхозности в 2026.

### Code highlighting
Для кода в демо-сниппетах (если появятся) — `shiki` или `rehype-pretty-code` с темой **Vesper**, **Ayu Dark** или **Vitesse Dark**. Не Dracula, не Monokai — клише.

### Что НЕ делать
- Стоковые фото людей в костюмах.
- 3D-иллюстрации.
- «Летающие» элементы, parallax.
- Много текста в Hero.
- Больше двух шрифтов.
- Карусели отзывов с автопрокруткой.
